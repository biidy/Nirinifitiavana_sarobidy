## 1. Analyse Exploratoire des Données (EDA)
**Contexte :** Comprendre les données brutes, les nettoyer et formuler des hypothèses solides avant d'appliquer des modèles de machine learning plus complexes.

🔗 **[📦 Code Source (GitHub)](https://github.com/votre-username/eda-credit-card-marketing) | 📊 [Notebook Jupyter](https://github.com/votre-username/eda-credit-card-marketing/blob/main/notebooks/eda.ipynb) | 🚀 [Dashboard Interactif (Streamlit/Shiny)](https://votre-application.streamlit.app)**

* **Objectifs :** Comprendre la distribution des données, identifier les variables importantes et déployer des visualisations interactives pour les parties prenantes.
* **Problématiques et Datasets :**
  * **Banque :** Analyse des transactions pour identifier des schémas de fraude. [Dataset Kaggle](https://www.kaggle.com/datasets/mlg-ulb/creditcardfraud)
  * **Marketing :** Identification des segments clients les plus rentables suite aux campagnes. [Dataset Kaggle](https://www.kaggle.com/datasets/rodsaldanha/arketing-campaign/data)
* **Technologies :** Python, Pandas, NumPy, Matplotlib, Seaborn, Shiny / Streamlit / Dash.
* **Compétences acquises :** Manipulation de grands volumes de données, gestion des valeurs manquantes et doublons, statistique univariée/multivariée, et création de tableaux de bord interactifs.

## 2. A/B Testing
**Contexte :** Expérimentation contrôlée pour comparer deux versions d'un produit (très populaire en e-commerce et marketing digital) afin de déterminer la plus performante.

🔗 **[📦 Code Source (GitHub)](https://github.com/votre-username/ab-testing-landing-page) | 📊 [Notebook d'Analyse Statistique](https://github.com/votre-username/ab-testing-landing-page/blob/main/notebook.ipynb)**

* **Objectifs :** Mettre en place des groupes de contrôle/test et analyser les résultats statistiquement pour valider la version gagnante.
* **Problématiques et Datasets :**
  * **Marketing :** Évaluation du taux de conversion entre deux versions d'une landing page. [Dataset Kaggle](https://www.kaggle.com/datasets/zhangluyuan/ab-testing)
* **Technologies :** Python, Pandas, SciPy (tests statistiques), Matplotlib, Seaborn.
* **Compétences acquises :** Tests statistiques (T-test, Chi-deux), analyse de résultats, aide à la prise de décision via la data visualisation.

🔗 **[📦 Code Source (GitHub)](https://github.com/votre-username/customer-segmentation) | 📊 [Notebook d'Analyse des Clusters](https://github.com/votre-username/customer-segmentation/blob/main/clustering.ipynb)**

## 4. Modélisation de Régression
**Contexte :** Prédiction de valeurs continues en fonction d'un ensemble de variables explicatives.

🔗 **[📦 Code Source (GitHub)](https://github.com/votre-username/sales-house-price-regression) | 📊 [Notebook d'Entraînement](https://github.com/votre-username/sales-house-price-regression/blob/main/train.ipynb) | 🚀 [Lien vers l'API Déployée](https://votre-api-regression.onrender.com)**

* **Objectifs :** Construire, comparer et optimiser les performances de différents algorithmes de régression, puis les déployer.
* **Problématiques et Datasets :**
  * **Marketing :** Prédiction des ventes futures. [Dataset Kaggle](https://www.kaggle.com/datasets/yasserh/advertising-sales-dataset/data)
  * **Immobilier :** Prédiction du prix de vente d'une maison selon ses caractéristiques. [Dataset Kaggle](https://www.kaggle.com/c/house-prices-advanced-regression-techniques)
* **Technologies :** Python, Scikit-learn, XGBoost, LightGBM, CatBoost, Flask / FastAPI, Docker.
* **Compétences acquises :** Régularisation (Ridge, Lasso, ElasticNet), méthodes ensemblistes (Random Forest, AdaBoost), création d'API de prédiction et conteneurisation pour le cloud.

## 5. Classification et Modélisation Prédictive
**Contexte :** Création d'algorithmes capables de catégoriser des données et de prédire des événements probables, une compétence clé dans les secteurs bancaires et médicaux.

🔗 **[📦 Code Source (GitHub)](https://github.com/votre-username/predictive-classification-models) | 📊 [Notebook d'Évaluation](https://github.com/votre-username/predictive-classification-models/blob/main/evaluation.ipynb) | 🚀 [Lien vers l'API Déployée](https://votre-api-classification.onrender.com)**

* **Objectifs :** Développer et comparer les métriques de performance de modèles de classification sur des cas d'usage critiques.
* **Problématiques et Datasets :**
  * **Banque (Défaut) :** Probabilité qu'un client fasse défaut sur son prêt. [Dataset Kaggle](https://www.kaggle.com/datasets/itsmesunil/bank-loan-modelling)
  * **Banque (Risque) :** Évaluation du risque de crédit. [Dataset Kaggle](https://www.kaggle.com/datasets/laotse/credit-risk-dataset)
  * **Santé :** Prédiction du risque de réadmission à l'hôpital. [Dataset Kaggle](https://www.kaggle.com/datasets/redwankarimsony/heart-disease-data)
  * **Télécom :** Prédiction de l'attrition client (Churn). [Dataset Kaggle](https://www.kaggle.com/datasets/blastchar/telco-customer-churn)
* **Technologies :** Python, Scikit-learn, GBM, XGBoost, Flask / FastAPI, Docker.
* **Compétences acquises :** Régression logistique, SVM, KNN, Naive Bayes, Ensemble Learning, et mise en production d'API prédictives scalables.
